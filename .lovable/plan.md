

## Set Up Lovable Cloud Database for Persistent Data

This will migrate the app from in-memory React state to a Supabase database so data persists across sessions.

### Database Tables to Create

**shippers** — all columns from the `Shipper` type (company_name, address, city, state, zip, phone, email, sales_stage, credit_limit, payment_terms, notes, shipping_manager_name, direct_phone, estimated_monthly_loads, last_contact_date, next_follow_up, created_at)

**contacts** — shipper_id FK, first_name, last_name, title, phone, email, is_primary

**lanes** — shipper_id FK, origin, destination, rate, equipment_type, notes

**follow_ups** — shipper_id FK, date, notes, completed

**activities** — entity_id, entity_type, type, description, timestamp, "user"

**carriers** — all columns from `Carrier` type (company_name, mc_number, dot_number, address, city, state, zip, phone, email, equipment_types (text[]), insurance_expiry, insurance_provider, packet_status, factoring_company, factoring_remit_to, w9_uploaded, insurance_cert_uploaded, carrier_packet_uploaded, notes, created_at)

**loads** — load_number, shipper_id FK, carrier_id FK (nullable), origin, destination, pickup_date, delivery_date, shipper_rate, carrier_rate, weight, equipment_type, status, pod_uploaded, invoice_number, invoice_date, invoice_amount, payment_status, notes, created_at

**contracts** — type, status, entity_id, entity_type, load_id, title, terms, signed_by_name, signed_at, created_at, expires_at

**outbound_calls** — shipper_id FK, contact_name, contact_title, direct_phone, email, call_attempt_number, call_date, call_outcome, pain_point, notes, next_step, next_follow_up_date, assigned_sales_rep, created_at

**sales_tasks** — shipper_id FK, type, title, description, due_date, completed, completed_at, template_id, cadence_day, created_at

**email_templates** — name, subject, body, created_at

**stage_change_logs** — shipper_id FK, from_stage, to_stage, changed_at, changed_by

### Code Changes

1. **Enable Lovable Cloud** — Connect the project to Supabase and run migrations for all 12 tables above. RLS disabled initially (no auth yet).

2. **Create `src/integrations/supabase/` client and types** — Auto-generated by Lovable Cloud.

3. **Create custom hooks** (`src/hooks/`) for each entity using `@tanstack/react-query`:
   - `useShippers` — fetches, creates, updates shippers
   - `useCarriers` — fetches, creates, updates carriers
   - `useLoads` — fetches, creates, updates loads
   - `useContracts` — fetches, creates, updates contracts
   - Plus hooks for contacts, lanes, follow_ups, activities, outbound_calls, sales_tasks, email_templates, stage_change_logs

4. **Refactor `AppContext.tsx`** — Replace `useState` with data from the Supabase hooks. Expose mutation functions instead of raw `setState`. Keep the same interface shape so pages don't break.

5. **Update all pages** that call `setShippers(prev => ...)`, `setCarriers(prev => ...)`, etc. to use the mutation functions which write to Supabase and invalidate the query cache.

6. **Remove `src/data/mockData.ts`** data imports from AppContext (keep label mappings).

### What This Means

- All shippers, carriers, loads, and contracts you create will be saved to a real database
- Data survives page refreshes, deploys, and browser sessions
- No authentication is added yet — this is a single-user persistence layer
